## Overview

Tabs allow you to present multiple variations of content in a compact, interactive format. They're particularly useful for displaying platform-specific instructions, code examples in different languages, or alternative approaches to the same task.

## Basic Usage

Wrap individual `Tab` components inside a `TabGroup` to create a tabbed interface.

## Syntax

```mdx
<TabGroup>
  <Tab title="First Tab">
    Content for the first tab
  </Tab>
  
  <Tab title="Second Tab">
    Content for the second tab
  </Tab>
</TabGroup>
```

## Props

### TabGroup

<ParameterField name="className" type="string">
Optional CSS class name for custom styling
</ParameterField>

### Tab

<ParameterField name="title" type="string">
Display text shown in the tab header
</ParameterField>

## Behavior

Tabs operate as a single-selection component. Clicking a tab header displays its content while hiding all other tab content. The first tab is selected by default on page load.

## Example: Code Snippets

Use tabs to show the same operation across different programming languages:

<TabGroup>
  <Tab title="JavaScript">
  ```javascript
  fetch('https://api.example.com/data', {
        method: 'GET',
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY'
        }
      })
        .then(response => response.json())
        .then(data => console.log(data));
  ```
  </Tab>
  <Tab title="Python">
  ```python
  import requests

  response = requests.get(
      'https://api.example.com/data',
      headers={'Authorization': 'Bearer YOUR_API_KEY'}
  )
  print(response.json())
  ```
  </Tab>
  <Tab title="cURL">
  ```bash
  curl -X GET 'https://api.example.com/data' \
    -H 'Authorization: Bearer YOUR_API_KEY'
  ```
  </Tab>
</TabGroup>

## Example: Rich Content

Tabs support any MDX content including callouts, steps, and nested components:

<TabGroup>
  <Tab title="Installation">
  <Steps>
    <Step title="Install package">
  Run the installation command for your package manager:

          ```bash
          npm install @example/sdk
          ```
    </Step>
    <Step title="Import SDK">
  Import the SDK in your application:

          ```javascript
          import { Client } from '@example/sdk';
          ```
    </Step>
  </Steps>
  </Tab>
  <Tab title="Configuration">
  <Callout type="warning">
  Never commit API keys to version control
  </Callout>

  Create a `.env` file with your credentials:

  ```bash
  API_KEY=your_api_key_here
  API_SECRET=your_secret_here
  ```
  </Tab>
  <Tab title="Usage">
  Initialize the client and make your first request:

  ```javascript
  const client = new Client({
    apiKey: process.env.API_KEY
  });

  const result = await client.getData();
  ```
  </Tab>
</TabGroup>

## Best Practices

**Use tabs when:**

- Showing platform-specific instructions (macOS, Windows, Linux)
- Providing code examples in multiple languages
- Presenting alternative approaches to the same task
- Displaying different configuration formats

**Avoid tabs when:**

- Content needs to be visible simultaneously for comparison
- Sequential steps must be followed in order (use Steps instead)
- Only two options exist and both are brief (consider simple headings)
- Content should be scannable without interaction

<Callout type="tip">
Keep tab titles short and descriptive. Users should understand the content difference without clicking.
</Callout>
