## Overview

Cards organize content into visual containers that highlight important information, features, or navigation options. Use cards to create scannable layouts that guide users to specific resources or explain key concepts.

## Basic Usage

Create a simple card with a title, description, and content.

<TabGroup>
  <Tab title="Preview">
    <Card title="Simple Card" description="This is a basic card with title and description">
      Basic card content goes here. This card uses the default attributes.
    </Card>
  </Tab>
  <Tab title="Syntax">
    ```mdx
    &lt;Card title="Simple Card" description="This is a basic card with title and description"&gt;
      Basic card content goes here. This card uses the default attributes.
    &lt;/Card&gt;
    ```
  </Tab>
</TabGroup>

## Card Attributes

<ParameterField name="title" type="string">
Main heading displayed at the top of the card.
</ParameterField>

<ParameterField name="description" type="string">
Subtitle text that appears below the title providing additional context.
</ParameterField>

<ParameterField name="icon" type="string">
Icon name from [Lucide React](https://lucide.dev) icon library. Displayed alongside the title when `showIcon` is true.
</ParameterField>

<ParameterField name="showIcon" type="boolean" default="true">
Controls icon visibility. Set to `false` to hide the icon even when specified.
</ParameterField>

<ParameterField name="horizontal" type="boolean" default="false">
Changes layout orientation. When `true`, displays icon on left with content on right.
</ParameterField>

<ParameterField name="href" type="string">
URL destination. Makes the entire card clickable and navigable. Supports both internal and external links.
</ParameterField>

## Adding Icons

Icons enhance visual hierarchy and help users quickly identify card content. All icons come from the Lucide React library.

<TabGroup>
  <Tab title="Preview">
    <Card title="Card with Icon" description="This card demonstrates the icon attribute" icon="star">
      This card includes an icon. The icon name should be from Lucide React.
    </Card>

    <Card title="Card without Icon" description="This card has showIcon set to false" icon="user">
      Even though an icon is specified, it won't be displayed because showIcon is false.
    </Card>
  </Tab>
  <Tab title="Syntax">
    ```mdx
    &lt;Card title="Card with Icon" description="This card demonstrates the icon attribute" icon="star"&gt;
      This card includes an icon. The icon name should be from Lucide React.
    &lt;/Card&gt;

    &lt;Card title="Card without Icon" description="This card has showIcon set to false" icon="user" showIcon={false}&gt;
      Even though an icon is specified, it won't be displayed because showIcon is false.
    &lt;/Card&gt;
    ```
  </Tab>
</TabGroup>

<Callout type="tip">
Browse available icons at [lucide.dev](https://lucide.dev) and use the exact icon name in lowercase with hyphens (e.g., `external-link`, `file-text`).
</Callout>

## Layout Options

Cards support two layout modes: vertical (default) and horizontal.

### Vertical Layout

The default layout stacks content from top to bottom with the icon above the title.

  </Tab>
  <Tab title="Syntax">
    ```mdx
    &lt;Card title="Vertical Card" description="This is the default vertical layout" icon="layers"&gt;
      This card uses the default vertical layout where content flows from top to bottom.
    ```

  </Tab>

Set `horizontal={true}` to display the icon on the left with content on the right, similar to callouts.

<Callout type="tip">
Use clickable cards for navigation to guides, API references, or external resources. They provide larger click targets than standard links.
</Callout>

## Card Groups

Use `CardGroup` to display multiple cards in a grid layout with consistent spacing.

<TabGroup>
  <Tab title="Preview">
    <CardGroup cols={2} gap="md">
      <Card title="Authentication" icon="key" href="/auth">
        Learn how to authenticate requests using API keys or JWT tokens.
      </Card>
      <Card title="Rate Limiting" icon="clock" href="/rate-limits">
        Understand rate limits and best practices for high-volume usage.
      </Card>
      <Card title="Error Handling" icon="alert-circle" href="/errors">
        Handle errors gracefully with proper status codes and messages.
      </Card>
      <Card title="Webhooks" icon="webhook" href="/webhooks">
        Set up webhooks to receive real-time event notifications.
      </Card>
    </CardGroup>
  </Tab>
  <Tab title="Syntax">
    ```mdx
    &lt;CardGroup cols={2}&gt;
      &lt;Card title="Authentication" icon="key" href="/auth"&gt;
        Learn how to authenticate requests using API keys or JWT tokens.
      &lt;/Card&gt;

      &lt;Card title="Rate Limiting" icon="clock" href="/rate-limits"&gt;
        Understand rate limits and best practices for high-volume usage.
      &lt;/Card&gt;

      &lt;Card title="Error Handling" icon="alert-circle" href="/errors"&gt;
        Handle errors gracefully with proper status codes and messages.
      &lt;/Card&gt;

      &lt;Card title="Webhooks" icon="webhook" href="/webhooks"&gt;
        Set up webhooks to receive real-time event notifications.
      &lt;/Card&gt;
    &lt;/CardGroup&gt;
    ```
  </Tab>
</TabGroup>

<Callout type="info">
The `cols` attribute on `CardGroup` controls the number of columns in the grid. Common values are 2 or 3 depending on your layout needs.
</Callout>

## Best Practices

<AccordionGroup>
  <Accordion title="When to use cards">
    - **Feature highlights** - Showcase key features or capabilities
    - **Navigation hubs** - Guide users to different documentation sections
    - **Quick start guides** - Present multiple getting started paths
    - **Resource collections** - Group related links or references
  </Accordion>
  <Accordion title="Writing effective card content">
    - Keep titles concise (2-5 words)
    - Write descriptions that clearly explain the card's purpose
    - Choose icons that visually represent the content
    - Use consistent card structures across similar content
    - Limit card content to essential information
  </Accordion>
  <Accordion title="Accessibility considerations">
    - Ensure linked cards have descriptive titles, not generic "Click here"
    - Choose icons that reinforce meaning, not as the sole indicator
    - Maintain sufficient color contrast in custom styling
    - Test keyboard navigation for clickable cards
  </Accordion>
</AccordionGroup>
