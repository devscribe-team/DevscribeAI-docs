## Overview 

The Expand component creates collapsible sections that hide optional content until users need it. Use it to reduce visual clutter while keeping detailed information accessible.

## Basic Usage

The simplest implementation uses default settings with the content collapsed by default.

<ParameterField name="title" type="string" default="Details">

</ParameterField>

<ParameterField name="defaultOpen" type="boolean" default="false">
Controls initial state. Set to `true` to display content expanded on page load.

## Best Practices

<AccordionGroup>
  <Accordion title="When to use Expand">
  - **Optional details** - Advanced configuration, technical specifications, or edge cases
  - **Verbose examples** - Long code samples or detailed output that clutters the page
  - **Nested API properties** - Object hierarchies with multiple levels (use with ResponseField)
  - **Troubleshooting steps** - Error resolution that most users won't need
  </Accordion>
  <Accordion title="When not to use Expand">
  - **Critical information** - Prerequisites, warnings, or essential setup steps
  - **Primary instructions** - Main workflow steps should remain visible
  - **Short content** - Content under 3-4 lines doesn't justify hiding
  - **Navigation** - Use Tabs or Accordions for alternative paths or categories
  </Accordion>
  <Accordion title="Writing effective titles">
  - Use descriptive, actionable titles: "Advanced Configuration Options" not "More Info"
  - Indicate content type: "Example Response", "Troubleshooting Steps", "Code Sample"
  - Keep titles concise: 2-6 words maximum
  - Use parallel structure when multiple expands appear together
  </Accordion>
</AccordionGroup>

### Troubleshooting Sections

Group related solutions under descriptive expand titles.

<Expand title="Connection timeout errors">
**Cause**: Network latency or firewall blocking requests.

**Solutions**:
1. Increase timeout in configuration: `timeout: 30000`
2. Check firewall rules for ports 80 and 443
3. Verify DNS resolution: `nslookup api.example.com`
</Expand>

<Expand title="Authentication failures">
**Cause**: Invalid or expired API key.

**Solutions**:
1. Regenerate API key in dashboard settings
2. Verify key format: `Bearer sk_live_xxxxxxxxxxxx`
3. Check key permissions match required scopes
</Expand>
