## Overview

The Steps component displays sequential procedures with automatic numbering and visual progression indicators. Each step can contain text, code examples, callouts, and other Devscribe components.

## Component Structure

### Steps Container

The `<Steps>` wrapper creates the numbered sequence container.

```markdown
<Steps>
  <!-- Step components go here -->
</Steps>
```

### Individual Step

Each `<Step>` requires a `title` attribute and can contain any markdown or Devscribe components.

```markdown
<Step title="Your step title">
Step content goes here
</Step>
```

### Multiple Components

Combine text, code, and callouts within a single step for comprehensive instructions.

<Steps>
  <Step title="Implement Authentication">
Create the authentication middleware:

<CodeSnippet language="javascript" title="middleware/auth.js" highlight="3-5">
{`import jwt from 'jsonwebtoken'

export const authenticate = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1]
  if (!token) return res.status(401).json({ error: 'No token provided' })
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET)
    req.user = decoded
    next()
  } catch (error) {
    res.status(401).json({ error: 'Invalid token' })
  }
}`}
</CodeSnippet>

<Callout type="tip">
Store JWT secrets in environment variables, never in source code.
</Callout>

Apply the middleware to protected routes.
  </Step>
</Steps>

## Best Practices

### Use Clear, Action-Oriented Titles

Write step titles that describe the action users will take.

<CardGroup cols={2} gap="md">
  <Card title="Good" icon="check">
```markdown
<Step title="Install Dependencies">
<Step title="Configure Environment Variables">
<Step title="Deploy to Production">
```
  </Card>
  <Card title="Avoid" icon="x">
```markdown
<Step title="Dependencies">
<Step title="Environment">
<Step title="Production">
```
  </Card>
</CardGroup>

### Provide Context Before Code

Explain what the code does before showing it.

```markdown
<Step title="Create API Client">
Create a reusable API client that handles authentication and error handling:

<CodeSnippet language="typescript" title="lib/api-client.ts">
{`// Implementation here`}
</CodeSnippet>
</Step>
```
