---
title: "Ordered Lists"
description: "Create numbered sequential lists in your documentation"
---

Ordered lists display content as numbered items in sequential order. Use them for step-by-step instructions, ranked items, or any content where sequence matters.

## Basic usage

<TabGroup>
  <Tab title="Preview">
    <ol>
      <li>First item</li>
      <li>Second item</li>
      <li>Third item</li>
    </ol>
  </Tab>
  <Tab title="Syntax">
    <pre><code>{`1. First item
2. Second item
3. Third item`}</code></pre>
  </Tab>
</TabGroup>

<Callout type="info">
Markdown automatically handles numbering. You can write all items as `1.` and they'll render correctly.
</Callout>

## When to use ordered lists

Use ordered lists when:

- Sequence matters — steps must be completed in a specific order
- Ranking content — items are prioritized or ordered by importance
- Reference by number — users need to refer to specific steps
- Multi-step procedures — breaking down complex tasks into discrete actions

<Callout type="tip">
For procedures with additional context, code examples, or warnings, use the Steps component instead for better visual hierarchy.
</Callout>

## Multi-level ordered lists

Create nested ordered lists by indenting with 2-4 spaces:

<TabGroup>
  <Tab title="Preview">
    <ol>
      <li>
        First level item
      </li>
      <li>
        Another first level item
        <ol>
          <li>Nested item</li>
          <li>Another nested item</li>
        </ol>
      </li>
      <li>Back to first level</li>
    </ol>
  </Tab>
  <Tab title="Syntax">
    <pre><code>{`1. First level item
2. Another first level item
   1. Nested item
   2. Another nested item
3. Back to first level`}</code></pre>
  </Tab>
</TabGroup>

## Combining with other content

Ordered lists work seamlessly with other markdown elements:

<TabGroup>
  <Tab title="Preview">
    <ol>
      <li>Install the package</li>
      <li>Configure your environment variables</li>
      <li>Start the development server</li>
    </ol>

    <pre><code>npm install example-package</code></pre>
  </Tab>
  <Tab title="Syntax">
    <pre><code>{`1. Install the package

\`\`\`bash
npm install example-package
\`\`\`

2. Configure your environment variables
3. Start the development server`}</code></pre>
  </Tab>
</TabGroup>

<Callout type="warning">
Maintain consistent indentation when including code blocks or multi-paragraph content within list items. Use 3 spaces for proper alignment.
</Callout>

## Ordered lists vs Steps component

<CardGroup cols={2} gap="md">
  <Card title="Use ordered lists" icon="list-ol">
    <p>Simple sequential items</p>
    <p>Brief, single-line steps</p>
    <p>Minimal additional context</p>
    <p>Embedded within larger content</p>
  </Card>
  <Card title="Use Steps component" icon="stairs">
    <p>Complex multi-step procedures</p>
    <p>Steps requiring code examples</p>
    <p>Procedures with callouts or warnings</p>
    <p>Standalone tutorials or guides</p>
  </Card>
</CardGroup>
