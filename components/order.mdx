Ordered lists display content as numbered items in sequential order. Use them for step-by-step instructions, ranked items, or any content where sequence matters.

## Basic usage 

<TabGroup>
  <Tab title="Preview">
  1. First item
  2. Second item
  3. Third item
  </Tab>
  <Tab title="Syntax">
  ```markdown
    1. First item
    2. Second item
    3. Third item
  ```
  </Tab>
</TabGroup>

<Callout type="info">
Markdown automatically handles numbering. You can write all items as `1.` and they'll render correctly.
</Callout>

## When to use ordered lists

Use ordered lists when:

- **Sequence matters** - Steps must be completed in a specific order
- **Ranking content** - Items are prioritized or ordered by importance
- **Reference by number** - Users need to refer to specific steps
- **Multi-step procedures** - Breaking down complex tasks into discrete actions

<Callout type="tip">
For procedures with additional context, code examples, or warnings, use the Steps component instead for better visual hierarchy.
</Callout>

## Multi-level ordered lists

Create nested ordered lists by indenting with 2-4 spaces:

<TabGroup>
  <Tab title="Preview">
  1. First level item
  2. Another first level item
  3. Nested item
  4. Another nested item
  5. Back to first level
  </Tab>
  <Tab title="Syntax">
  ```markdown
    1. First level item
    2. Another first level item
       1. Nested item
       2. Another nested item
    3. Back to first level
  ```
  </Tab>
</TabGroup>

Ordered lists work seamlessly with other markdown elements:

<Callout type="warning">
Maintain consistent indentation when including code blocks or multi-paragraph content within list items. Use 3 spaces for proper alignment.
</Callout>

## Ordered lists vs Steps component

<CardGroup cols={2} gap="md">
  <Card title="Use ordered lists" icon="list-ol">
- Simple sequential items
  - Brief, single-line steps
  - Minimal additional context
  - Embedded within larger content
  </Card>
  <Card title="Use Steps component" icon="stairs">
- Complex multi-step procedures
  - Steps requiring code examples
  - Procedures with callouts or warnings
  - Standalone tutorials or guides
  </Card>
</CardGroup>
