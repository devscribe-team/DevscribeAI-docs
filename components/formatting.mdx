## Basic Text Formatting

Format text to emphasize important information and improve readability.

### Bold Text

Use double asterisks to make text **bold**:

```markdown
**bold text**
```

Use bold for:
- Important concepts or terms on first mention
- UI element names like button labels
- Critical warnings within sentences

### Italic Text

Use single asterisks to make text *italic*:

```markdown
*italic text*
```

Use italics for:
- Emphasis within sentences
- Variable names or placeholders in prose
- Book or article titles

### Inline Code

Use backticks to format text as `inline code`:

```markdown
`inline code`
```

Use inline code for:
- File names and paths
- Function and method names
- Command names and flags
- API endpoints
- Environment variables
- Code snippets within sentences

### Strikethrough

Use double tildes to add ~~strikethrough~~ formatting:

```markdown
~~strikethrough~~
```

Use strikethrough for:
- Deprecated features or methods
- Outdated information being replaced
- Showing corrections in change logs

## Combining Formats

Combine formatting styles for specific purposes:

- ***Bold and italic***: `***bold and italic***`
- **`Bold inline code`**: `**\`bold code\`**`
- *`Italic inline code`*: `*\`italic code\`*`

<Callout type="tip">
Avoid over-formatting text. Use formatting purposefully to guide readers to important information.
</Callout>

## Best Practices

- **Be consistent** - Use the same formatting for the same types of content throughout your documentation
- **Don't overuse** - Too much formatting reduces readability and diminishes emphasis
- **Use semantic meaning** - Choose formatting that matches the content's purpose (code vs emphasis vs importance)
- **Test readability** - Ensure formatted text remains legible on different screen sizes and themes

<Callout type="warning">
Avoid nesting multiple formatting styles excessively. Keep text formatting simple and purposeful.
</Callout>
