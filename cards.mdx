The Card component creates self-contained content blocks that help organize information and improve readability. Cards can include titles, descriptions, icons, and custom content.

## Properties

<ParameterField name="title" type="string" required>
  Card heading that appears at the top of the component.
</ParameterField>

<ParameterField name="description" type="string">
  Subtitle text that appears below the title for additional context.
</ParameterField>

<ParameterField name="icon" type="string">
  Icon name from the [Lucide React](https://lucide.dev/) icon library. Display controlled by `showIcon`.
</ParameterField>

<ParameterField name="showIcon" type="boolean" default="true">
  Controls icon visibility. Set to `false` to hide the icon even when `icon` is specified.
</ParameterField>

<ParameterField name="href" type="string">
  URL to navigate to when the card is clicked, converting the card into a clickable link.
</ParameterField>

## Common Patterns

### Navigation Cards

Use cards with `href` to create visual navigation menus for documentation sections.

```markdown
<Card title="Getting Started" icon="rocket" href="/quickstart">
  Complete setup in under 10 minutes with our quickstart guide.
</Card>
```

### Feature Highlights

Combine cards in groups to showcase product features or capabilities.

```markdown
<CardGroup cols={3}>
  <Card title="Fast" icon="zap">
    Optimized for performance with sub-second response times.
  </Card>

  <Card title="Reliable" icon="shield">
    99.9% uptime SLA with automatic failover.
  </Card>

  <Card title="Scalable" icon="trending-up">
    Handle millions of requests without configuration changes.
  </Card>
</CardGroup>
```

### Content Organization

Use cards to separate distinct topics or concepts within a single page.

```markdown
<Card title="Prerequisites" icon="list-checks">
  - Node.js 18 or higher
  - Valid API key
  - Basic JavaScript knowledge
</Card>
```
